1. Le besoin initial

La surveillance proactive des infrastructures est essentielle pour anticiper les pannes. Gatus est un outil de surveillance et de "healthcheck" open-source très léger. Le but de ce projet était de le déployer pour surveiller plusieurs services critiques et alerter l'équipe en temps réel en cas de dysfonctionnement.

2. Déploiement et Configuration

L'installation s'est faite de manière isolée et reproductible :

  • Conteneurisation : Déploiement de l'outil Gatus via docker-compose pour une gestion simplifiée.
  • Fichier YAML : Définition des endpoints à surveiller (HTTP, ICMP, DNS) directement dans le fichier de configuration config.yaml.
  • Conditions de succès : Configuration de règles précises (réponse HTTP 200, temps de réponse inférieur à 500ms, etc.) pour définir l'état de santé du service.
Dashboard Gatus affichant l'état des services monitorés

3. Alerting via Discord

Pour être averti instantanément lors de la chute d'un service (Downtime), une intégration par Webhook a été réalisée avec l'application Discord. Dès qu'un endpoint ne répond plus aux critères définis dans le YAML, Gatus envoie automatiquement une notification détaillée sur un canal privé dédié à l'équipe système.
