1. Présentation du projet

L'objectif de ce projet était de déployer une infrastructure robuste de type "3-Tiers" (Client, Serveur d'application, Serveur de base de données). L'utilisation de PostgreSQL couplé à pgAdmin offre une solution d'administration de bases de données performante, nécessitant une forte sécurisation des flux.

2. Installation et Sécurisation

La sécurité a été placée au centre du déploiement avec la mise en place d'un chiffrement SSL obligatoire :

  • Déploiement : Installation du moteur PostgreSQL et de l'interface d'administration pgAdmin.
  • Génération de certificats : Création de certificats auto-signés via OpenSSL (Clé privée, Certificat x509).
  • Configuration TLS : Paramétrage des fichiers de configuration PostgreSQL (postgresql.conf et pg_hba.conf) pour forcer les connexions chiffrées (hostssl).
Interface d'administration pgAdmin connectée en SSL

3. Résultats et Validation

L'accès à l'interface web de pgAdmin s'effectue désormais exclusivement via le protocole HTTPS. Les communications entre l'interface web et le serveur PostgreSQL sont totalement chiffrées, garantissant la confidentialité des requêtes SQL et des données transitant sur le réseau.
