1. Présentation du projet

Bernard Quizz est une application web de quiz multijoueur en temps réel développée intégralement de A à Z. Un hôte crée un salon, choisit un quiz et un mode de jeu, puis partage un code PIN à 6 chiffres. Les joueurs rejoignent depuis n'importe quel appareil disposant d'un navigateur — sans installation. Les questions s'affichent simultanément sur tous les écrans, les scores sont mis à jour en temps réel, et un podium animé avec confettis clôture chaque partie.

Ce projet couvre l'ensemble de la chaîne : conception, développement, sécurisation et déploiement en environnement de production sur Alwaysdata.

2. Stack technique & choix d'architecture

L'architecture suit un modèle MVC léger sans framework. Chaque fichier PHP gère la logique métier et la vue. La communication temps réel repose sur un fichier central api_live.php qui lit et écrit des fichiers JSON dans un dossier /sessions/, servant de bus de données partagé entre tous les clients.

Composant Technologie Justification
Backend PHP 8.4 Maîtrisé, natif sur Alwaysdata, adapté au web rapide
Base de données MariaDB 11.4 Inclus dans l'offre hébergeur, relationnel, structuré
Frontend JS Vanilla + Tailwind CSS Pas de dépendance lourde, performance maximale client
Temps réel Polling HTTP 1,5s WebSockets non disponibles en mutualisé — solution pragmatique
Sécurité mots de passe bcrypt — password_hash() Algorithme adaptatif recommandé par PHP et OWASP
Accès base de données PDO + requêtes préparées Protection native contre les injections SQL

3. Fonctionnalités développées

  • Authentification complète : Inscription, connexion, sessions PHP, hachage bcrypt — conforme OWASP
  • Système de rôles hiérarchiques : 4 niveaux — utilisateur, créateur, admin, fondateur — vérifiés à chaque appel via hasRole()
  • Gestion de quiz : Éditeur CRUD complet (titre, image, questions, 4 options, timer, réponse correcte), visibilité public/privé
  • Avatars multicouches (14 couches) : Personnalisation complète par z-index (peau, vêtements, cheveux, barbe, auras, effets visuels) — restrictions selon le niveau du compte
  • Tchat en temps réel : Messages avec raccourcis emoji déclenchant des effets sonores sur l'écran hôte, bulles flottantes
  • Joker 50/50 : Suppression de 2 mauvaises réponses, 1 fois par partie, membres uniquement
  • Système de progression : 5 rangs, 2 quêtes de déblocage d'effets, statistiques persistantes (parties, précision, podiums)
  • Awards de fin de partie : L'Éclair, La Tortue, Le Miraculé — calculés côté serveur
  • Panel d'administration : Gestion des rôles utilisateurs, accès aux quiz de tous — réservé admin/fondateur

5. Ce que ce projet m'a apporté

Bernard Quizz est le premier projet que j'ai mené de la conception au déploiement en production. Il m'a permis de consolider mes compétences en développement web full-stack (PHP, MariaDB, JS), en sécurité applicative (injection SQL, hachage, gestion des sessions) et en administration de l'hébergement (configuration Apache, HTTPS, .htaccess, variables d'environnement). La contrainte technique des WebSockets m'a forcé à trouver une solution pragmatique avec le polling HTTP — une approche que l'on retrouve souvent en production réelle avec des hébergements mutualisés.
